OGC API - Maps Conformance Test Suite

This executable test suite (ETS) verifies that an OGC API - Maps instance conforms to
https://docs.ogc.org/is/20-058/20-058.html [OGC API - Maps - Part 1: Core] (OGC 20-058) and related standards.
Conformance testing is a kind of "black box" testing that examines externally visible characteristics or behaviors of the SUT and is independent of any implementation details.

The following conformance levels are tested:

  • Core
  • Coordinate Reference Systems by Reference

The test run arguments are summarized in the following table. The Obligation descriptor can have the following values: M (mandatory), O (optional), or C (conditional).

Test run arguments
Name Value domain Obligation Description
iut URI M A URI that refers to the landing page of the SUT. Ampersand ('&') characters must be percent-encoded as '%26'.
mapFormat String O The format of the map images to be tested. Default: image/png
crs String O The coordinate reference system (CRS) to use for map requests. Default: EPSG:3857
